首页 踩坑 java

1. 忘记读到 buf 数组。

2. 压缩包下载确保流的关闭，否则压缩包无法打开。
3. 输出流可以 flush 刷新一下，确保文件全部写入。
4.Integer 在 controller 能接受 Long 数，但在 mapper 查询出问题。
5. 用 Map 接收 requestBody 时，若 Map 的值为 String，传过来的参数要带上双引号 ""。
6. 当前端既可以传 Integer 又可以传 Long，后端用 Long 接收。但当传 Integer 时，报错 Integer 转 Long 出问题。（解决：toString 后用 Long.valueOf）
7. 文件下载出现文件名为 response.bin，因为：中文名没有编码。
为什么压缩文件的时候没有指定编码，没有出现这个问题？
8.postman 传 json 空格不同问题。
空格、NBSP 造成的 JSON 解析失败问题
9. 带多层目录的 zip 下载没出现文件名重复的错误（之前实现最简单的多个文件压缩成一个压缩包，文件名重复报错）
10.springboot 的参数校验 validate，PathVariable 的 required=true 校验可能先于 @Notnull？
11. 用来校验路径参数不能为空，否则不填路径参数直接到其他的请求返回 404 了： @GetMapping(value = {"/userList/{subscriber}", "/userList"})
12.requestbod 定义的对象含有 Map<String, Object>。对象对应的类需要有无参构造和全参构造器。
13. String.chatAt() 获得的是一个 char 字符，可以直接用 int 表示，但为 ascill 码。
14.CMS 垃圾回收器，第 4 个阶段并发清理，和用户线程并发执行，这时用户线程不会
15.（疑问）尚硅谷 jvm2020 版 P198，视频说 jdk7 的 eden 区放不下，gc 后再将新对象放 eden 区；而 jdk8 是直接放 old 区。但我实践后 jdk8 和 jdk7 行为是一样的：都是直接放 eden 区。
